附树莓派4的USB接口说明（树莓派等开发板连接移动硬盘）

黑色或白色为usb2.0接口，蓝色为usb3.0

准备

  1. 查看当前linux是否支持移动硬盘的文件系统类型
    • 移动硬盘的文件类型一般为NTFS，一般linux不支持NTFS文件系统的。（我的树莓派连接ntfs文件系统的移动硬盘后，可以进行读写操作，但是使用chmod，chown，chgrp等命令不能修改文件或文件夹权限，需要配置/etc/fstab文件统一配置权限，如何配置权限，请在本节末尾查看）
    findmnt：查看文件系统的信息，使用方式参考findmnt --help

blkid：来查看有哪些文件系统，使用方法参考 blkid --help

lsblk：查看当前可用或挂载的硬盘的文件系统（lsblk -f）

cat /proc/filesystems：查看内核支持的文件系统
    执行完上述任一条命令，若发现自己的linux系统支持ntfs的，则跳过第2步
    **注意：**想要在linux和windows都可以使用，推荐使用ntfs（大磁盘容量）或FAT32（小磁盘容量）的文件系统的硬盘（可以自己格式化磁盘的文件系统类型）
  2. 如何让Linux系统挂载NTFS文件系统类型的硬盘或移动硬盘。
    • 方法一：在linux系统中，下载NTFS文件系统的驱动，支持NTFS文件系统
    1. 先安装GCC
    sudo apt-get install gcc
    1. 下载NTFS-3G插件：点击下载
    2. NTFS-3G插件安装操作步骤
    cd /usr/local/src
进入下载的压缩包的所在目录

tar -zxvf ntfs-3g_ntfsprogs-2016.2.22.tgz 
解压

cd ntfs-3g_ntfsprogs-2016.2.22
进入解压后的目录

./configure 
编译前准备

make
编译

make install
安装
    NTFS-3G插件安装完成之后，Linux系统就可以支持NTFS文件系统了。(Linux系统将NTFS文件系统类型识别为ntfs-3g)
  • 方法二：格式化移动硬盘格式
    1. 将磁盘格式化为linux系统支持的文件类型，如：xfs，ext系列等

  1. linux挂载和卸载移动硬盘的命令：

    • 挂载移动硬盘或其他设备
        # mount -t [文件类型] [设备节点] [挂载目录]
    mount -t ntfs-3g /dev/sda1 /var/data
    • 卸载外设备或U盘
        # umount -v /dev/sda1          通过设备名卸载  
    # umount -v /mnt/mymount/      通过挂载点卸载
    • 补充，卸载硬盘和U盘的命令为啥不一样的，推荐这个博客：Josephine's Blog

问题排查

  1. 若使用fdisk -l命令找不到对应磁盘信息。可能存在的问题有两个。（其他的情况需要自己再慢慢踩坑补充）
    • 插错USB口，如果是虚拟机更改设备USB接口为3.0及以上，如果是树莓派等，则改换usb3.0插口
    • 硬盘文件系统格式和linux不兼容，格式化硬盘

磁盘分区步骤

  1. 查看本地已经存在的硬盘信息
    sudo fdisk -l
  2. 为磁盘创建分区
    sudo fdisk /dev/sdb    #/dev/sdb:是你移动硬盘的设备名称 --- 此命令进入磁盘分区
# 操作命令
 p：p命令查看打印当前磁盘的分区详细情况
 m：打印出help帮助信息
 n：创建分区（最多只能创建4个分区）
 # 根据提示信息进行分区操作
 d：删除一个分区
 w：保存内存中对磁盘分区的操作

 partprobe /dev/sda1：将分区信息写入内核中（忘记命令，直接输入part，会有提示）
 mkfs：命令格式化分区 （mkfs.xfs /dev/sdc1)---mkfs后面的.xfs为格式化的类型
 apt-get install xfsprogs：用于安装mkfs
    创建分区操作（图示）

开机自动挂载命令的配置

 # 永久自动挂载方式
 vim /etc/fstab
 # 首先查看UUID
 blkid
 # 将 /dev/vdb1 的 UUID 复制出来，然后写入到/etc/fstab中去
 echo "PARTUUID=88511c72-01 /home/pi/data ntfs defaults 0 0" >> /etc/fstab
 # 或者直接编辑 /etc/fstab
PARTUUID=88511c72-01 /home/pi/data ntfs defaults 0 0
要挂载的设备分区设备号 挂载的目录 文件系统类型 挂载选项 是否备份 是否检测
  • 注意1：文件挂载后，普通用户可能存在没有权限，普通的chrgp,chmod,chown等命令修改权限无效，则可以直接修改 < /etc/fstab > 参考链接 : 点击此处
  • 注意2：linux挂载ntfs文件系统，不能修改文件或文件夹权限，root用户也不行，原因和问题推荐jack_孟博主的文章：点击此处
